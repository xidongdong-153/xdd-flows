# 仓库文档初始化清单

这份清单用于新仓库建起来以后，快速补齐最小可用文档。

## 先建哪些文件

- [ ] `README.md`
- [ ] `AGENTS.md`
- [ ] `docs/architecture.md`
- [ ] `docs/development.md`

## 按复杂度再补

- [ ] `docs/modules/`
- [ ] `docs/api/`
- [ ] 其他项目专用说明

## 每份文档至少写什么

写这些说明性文档前，先看 [[01-规范/说明性文案规范]]。

### `README.md`

- 项目用于什么
- 仓库里主要有哪些包或目录
- 怎么启动
- 常用命令有哪些

### `AGENTS.md`

- Agent 先读哪些文件
- 哪些规则必须执行
- 哪些目录有特殊要求
- 文案、注释、测试、提交时怎么做

### `docs/architecture.md`

- 当前目录结构
- 每个核心目录放什么
- 主要模块之间怎么接
- 修改某类功能先去哪里看

### `docs/development.md`

- 开发命令
- 构建命令
- 测试命令
- 环境变量
- 常见调试入口

## 推荐顺序

1. 先把 `README.md` 和 `AGENTS.md` 写到能开工。
2. 再补 `docs/architecture.md` 和 `docs/development.md`。
3. 补文档时统一按 [[01-规范/说明性文案规范]] 调整文案。
4. 项目变复杂后，再拆 `docs/modules` 和 `docs/api`。
