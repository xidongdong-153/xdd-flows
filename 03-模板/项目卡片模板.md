## 文案规则

- README、`docs/`、JSDoc、代码注释、错误提示、提示词统一按 `[[01-规范/说明性文案规范]]` 来写
- 文案默认写中文
- 只写当前实现，不写历史过程和旧方案对比

## 输出规则

| 项目 | 要求 |
| --- | --- |
| 结论顺序 | 先给结论，再给修补方案 |
| 解释方式 | 不复读背景，直接用中文大白话说明 |
| 结构形式 | 多项内容优先用 Markdown 表格 |
| 等级展示 | 不写 `P0`、`P1` 这类等级 |
| 结尾收口 | 结束时明确说明这次用到的 skill |

## Agent 启动说明

请先阅读以下内容，再开始实现：

1. 仓库根目录 `AGENTS.md`
2. `README.md`
3. `docs/architecture.md`
4. `docs/development.md`
5. `[[01-规范/个人开发偏好]]`
6. `[[01-规范/开发技能调用规则]]`
7. `[[01-规范/说明性文案规范]]`
8. 如果任务涉及前端界面，先看 `[[01-规范/前端设计技能]]`
9. 当前任务直接相关的模块文档

执行要求：

- 先按已有目录和命名方式实现
- 技术选型默认先按个人开发偏好判断
- React / Next.js 任务先按开发技能调用规则使用 `vercel-react-best-practices`
- ElysiaJS 后端任务先按开发技能调用规则使用 `elysiajs`
- 说明性文案统一按当前实现写
- 涉及 README、`docs/`、注释、错误提示或提示词时，先按文案规范整理
- 输出先给结论和修补方案，多项内容优先用 Markdown 表格
- 不写汇报式背景复述，不显示 `P0`、`P1` 这类等级
- 结束时明确说明这次用到的 skill
- 改目录、改流程、改接口时同步更新文档
