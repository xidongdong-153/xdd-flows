# 个人开发偏好

这份笔记用于统一记录你的常用技术栈、实现倾向和主题方向。
新项目启动时、写项目卡片时、给 Agent 补上下文时，都先以这里为准。

## 技术栈偏好

| 类别   | 偏好                          |
| ---- | --------------------------- |
| 开发方向 | TypeScript 全栈开发             |
| 后端风格 | 偏好 ElysiaJS 这类简约、舒适、类型安全的风格 |
| 数据库  | PostgreSQL                  |
| 前端框架 | React、TanStack              |
| 全栈框架 | Next.js                     |

## 实现倾向

| 项目 | 偏好 |
| --- | --- |
| 代码感觉 | 简约、清爽、可直接维护 |
| 类型要求 | 优先保证类型安全 |
| 后端体验 | 喜欢接口层和类型定义都比较顺手、负担不重的方案 |
| 栈选择 | 前后端能用 TypeScript 打通时优先考虑打通 |

## 视觉偏好

| 项目 | 偏好 |
| --- | --- |
| 主题方向 | 偏好 Catppuccin 这类主题 |
| 调色参考 | [Catppuccin Palette](https://catppuccin.com/palette/) |
| 整体感觉 | 柔和、舒服、层次清楚，不要刺眼 |

## 使用方式

| 场景 | 怎么用 |
| --- | --- |
| 新项目建档 | 把这份偏好挂进项目卡片和 Agent 启动说明 |
| 技术选型 | 默认先按这里的技术栈偏好判断 |
| 前端设计 | 做界面时，把这里的主题偏好和 [[前端设计技能]] 一起看 |
| Agent 启动 | 如果没有单独例外，默认沿用这里的技术和主题偏好 |
