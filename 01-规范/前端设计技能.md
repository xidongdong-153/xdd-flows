# 前端设计技能

这份笔记用于统一记录你现在常用的前端设计 skill。
这类内容适合单独建一页，不建议塞进普通规范正文里。原因很简单：skill 数量多、组合关系多、后面还会继续加，单独成页更方便长期维护和快速查找。

## 技能出处与安装位置

| 资源族 | 包含 skill | 技能出处 | 安装位置 |
| --- | --- | --- | --- |
| Anthropic 原版设计技能 | `frontend-design` | [Anthropic frontend-design](https://github.com/anthropics/skills/tree/main/skills/frontend-design) | 当前统一安装在 `~/.agents/skills`，`~/.codex/skills` 和 `~/.claude/skills` 通过软链接接入 |
| Impeccable 设计技能体系 | `teach-impeccable`、`arrange`、`typeset`、`colorize`、`polish`、`normalize`、`bolder`、`quieter`、`clarify`、`onboard`、`animate`、`delight`、`distill`、`extract`、`audit`、`critique`、`optimize`、`overdrive`、`adapt`、`harden` | [Impeccable](https://impeccable.style/) | 官网给出的安装方式是 `npx skills add pbakaus/impeccable` |
| UI/UX Pro Max | `ui-ux-pro-max` | [nextlevelbuilder/ui-ux-pro-max-skill](https://github.com/nextlevelbuilder/ui-ux-pro-max-skill) | 仓库给出的 Codex 安装方式是先执行 `npm install -g uipro-cli`，再执行 `uipro init --ai codex` |
| Codex Front End 技能 | `frontend-skill` | 当前 Codex Front End 技能 | 当前统一安装在 `~/.agents/skills`，`~/.codex/skills` 和 `~/.claude/skills` 通过软链接接入 |

## 怎么用这份表

| 场景 | 建议做法 |
| --- | --- |
| 新项目要做界面 | 先看“推荐调用顺序” |
| 已经有页面，要定向优化 | 直接按“按用途速查”挑 skill |
| 想记住自己常用资源 | 先看“常用主 skill”，再看“收尾和补强” |
| 要给 Agent 一个稳定入口 | 在项目卡片或 `AGENTS.md` 里挂这份笔记 |

## 常用主 skill

| skill | 作用 | 什么时候先用 | 出处 |
| --- | --- | --- | --- |
| `teach-impeccable` | 记录项目设计上下文，给后续设计 skill 提供统一前提 | 新项目还没有明确设计上下文时 | Impeccable |
| `frontend-design` | 做整体界面设计方向和高质量前端实现 | 只要任务是页面、组件、后台界面、视觉改版 | Anthropic frontend-design |
| `frontend-skill` | 做更强视觉表达的页面、站点、落地页、Demo、原型 | 任务强调页面气质、视觉冲击、品牌表达时 | Codex Front End 技能 |
| `ui-ux-pro-max` | 快速查设计系统、风格、配色、字体和模式 | 想先找设计方向、设计系统和风格参考时 | UI/UX Pro Max |

## 按用途速查

| 类别    | skill       | 用途                   |
| ----- | ----------- | -------------------- |
| 布局与适配 | `arrange`   | 调整布局、间距、节奏和层级        |
| 布局与适配 | `adapt`     | 处理移动端、平板、桌面端和不同使用场景  |
| 字体与颜色 | `typeset`   | 调整字体、字号、层级和可读性       |
| 字体与颜色 | `colorize`  | 给界面补颜色和层次            |
| 风格增强  | `bolder`    | 让设计更有冲击力             |
| 风格增强  | `quieter`   | 把过强的设计压回更稳的状态        |
| 风格增强  | `delight`   | 加一点记忆点和小惊喜           |
| 风格增强  | `distill`   | 去掉多余复杂度，让页面更干净       |
| 动效与体验 | `animate`   | 补充有意义的动效和微交互         |
| 动效与体验 | `overdrive` | 做技术感更强、更出彩的前端效果      |
| 文案与引导 | `clarify`   | 调整按钮文案、提示语、错误说明和微文案  |
| 文案与引导 | `onboard`   | 做引导流程、空状态和首次使用体验     |
| 设计系统  | `normalize` | 统一样式，往设计系统收拢         |
| 设计系统  | `extract`   | 提炼可复用组件、样式变量和模式      |
| 检查与评估 | `critique`  | 从体验和视觉角度做设计评审        |
| 检查与评估 | `audit`     | 从可访问性、性能、响应式和主题做全面检查 |
| 收尾与上线 | `polish`    | 上线前做最后一轮细节收口         |
| 收尾与上线 | `harden`    | 补错误态、极端文本、i18n 和边界情况 |
| 收尾与上线 | `optimize`  | 优化加载、渲染、包体积和运行性能     |

## 推荐调用顺序

| 阶段 | 先用什么 | 后用什么 |
| --- | --- | --- |
| 新项目做界面 | `teach-impeccable` | `frontend-design` 或 `frontend-skill` |
| 先找风格方向 | `ui-ux-pro-max` | `frontend-design` |
| 页面已经有了，要调布局 | `frontend-design` | `arrange`、`adapt` |
| 页面已经有了，要调视觉 | `frontend-design` | `typeset`、`colorize`、`bolder`、`quieter`、`distill` |
| 页面已经有了，要调体验 | `frontend-design` | `clarify`、`onboard`、`animate` |
| 准备做系统化整理 | `frontend-design` | `normalize`、`extract` |
| 准备上线前收尾 | `critique` 或 `audit` | `polish`、`harden`、`optimize` |
| 想做一处亮点效果 | `frontend-design` | `overdrive` |

## 记忆方法

| 你要记什么 | 记法 |
| --- | --- |
| 总入口 | `frontend-design` |
| 先补上下文 | `teach-impeccable` |
| 先找设计系统 | `ui-ux-pro-max` |
| 做强视觉页面 | `frontend-skill` |
| 调布局 | `arrange` |
| 调字体 | `typeset` |
| 调颜色 | `colorize` |
| 调文案 | `clarify` |
| 做动效 | `animate` |
| 做收尾 | `polish` |
| 做检查 | `critique`、`audit` |
| 做上线补强 | `harden`、`optimize` |

## 使用提醒

| 提醒 | 说明 |
| --- | --- |
| 很多设计 skill 都依赖 `frontend-design` | 不要一上来只调局部 skill，先确认整体方向 |
| 没有设计上下文时先补上下文 | 新项目先跑 `teach-impeccable` 更稳 |
| `overdrive` 不适合乱用 | 只在需要做亮点效果时再上 |
| `audit` 和 `critique` 适合收尾前做 | 一个偏检查面，一个偏体验判断 |
| 本地安装位置按统一规则看 | 本地 skill 的实际维护目录统一看 `~/.agents/skills`，其他入口按软链接处理 |
