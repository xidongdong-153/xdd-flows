# 开发技能调用规则

这份笔记用于统一记录后端开发和 React/Next.js 开发时，哪些 skill 需要优先调用。
只要项目任务命中这些场景，就按这里的顺序走，不临时想一套。

## 核心规则

| 场景 | 先调用什么 | 什么时候用 |
| --- | --- | --- |
| ElysiaJS 后端开发 | `elysiajs` | 新增、修改、排查 ElysiaJS 路由、校验、认证、插件、集成、测试、部署 |
| React / Next.js 开发 | `vercel-react-best-practices` | 写 React 组件、Next.js 页面、做数据获取、重构、性能优化、代码评审 |

## ElysiaJS 调用规则

| 项目 | 说明 |
| --- | --- |
| skill | `elysiajs` |
| 适用任务 | ElysiaJS 路由、处理器、服务端结构、TypeBox 校验、认证、插件、Prisma/外部服务集成、WebSocket、测试、部署 |
| 参考入口 | [elysiajs.com/llms.txt](https://elysiajs.com/llms.txt) |
| 默认倾向 | 简约、舒适、类型安全、按功能拆目录 |
| 工作方式 | 先按 ElysiaJS 习惯整理路由、校验和服务，再继续实现业务 |

## React / Next.js 调用规则

| 项目 | 说明 |
| --- | --- |
| skill | `vercel-react-best-practices` |
| 适用任务 | React 组件、Next.js 页面、服务端渲染、客户端取数、重构、性能优化、代码评审 |
| 关注重点 | 消除 waterfall、控制 bundle、优化服务端和客户端渲染、减少不必要重渲染 |
| 默认倾向 | React 和 Next.js 实现优先考虑性能、结构清楚、类型安全 |
| 工作方式 | 写实现前先按规则看取数方式、组件边界和性能风险，再动代码 |

## 推荐顺序

| 任务类型 | 建议顺序                                                                        |
| --- | --- |
| 纯 ElysiaJS 后端任务 | `个人开发偏好 -> elysiajs -> 当前项目文档 -> 实现`                                        |
| React / Next.js 一般开发 | `个人开发偏好 -> vercel-react-best-practices -> 当前项目文档 -> 实现`                     |
| React / Next.js 界面任务 | `个人开发偏好 -> 前端设计技能 -> vercel-react-best-practices -> 当前项目文档 -> 实现`           |
| 全栈任务 | `个人开发偏好 -> 先按任务拆分调用 elysiajs 和 vercel-react-best-practices -> 当前项目文档 -> 实现` |

## 使用提醒

| 提醒 | 说明 |
| --- | --- |
| 后端任务别跳过 `elysiajs` | 只要后端核心是 ElysiaJS，就先按它的规则看一遍 |
| React / Next.js 不只看界面 | 除了设计，还要用 `vercel-react-best-practices` 控性能和数据流 |
| 设计和实现要分开看 | 界面风格走前端设计 skill，组件和性能走 React 最佳实践 skill |
