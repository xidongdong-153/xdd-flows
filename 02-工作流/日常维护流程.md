# 日常维护流程

这份流程用于项目已经开始后，怎么把仓库文档和 Obsidian 项目卡片一直维护在同一套节奏里。

## 平时怎么做

1. 开工前先看项目卡片，确认当前阶段、当前任务和文档入口。
2. 目录、命令、模块关系有变化时，先改仓库文档，再改项目卡片。
3. 新增重要模块时，在 `docs/modules` 补模块说明。
4. 新增对外接口或联调接口时，在 `docs/api` 补接口说明。
5. 每次阶段变化后，更新项目卡片里的 `status`、`stage`、`doc_status`、`agent_ready`。

## 每次收尾时检查

- 项目卡片里的仓库路径还是最新的
- 当前技术栈和目录约定没有过期
- Agent 启动说明还能直接用
- 仓库基础文档没有落后于当前实现

## 归档规则

- 项目暂停但后面还会继续，`status` 改成 `暂停`。
- 项目已经结束，`status` 改成 `归档`，再移动到 `90-归档`。
- 归档前补一段当前结论，方便以后回来时快速接上。
